# 固件能力说明

## 硬件平台

- 芯片：ESP32-C3
- 通信：USB 串口 (115200 baud) + BLE (NimBLE)
- LED：三色（绿/黄/红），有源低电平

## GPIO 引脚

| GPIO | 颜色 | 说明 |
|------|------|------|
| 0 | 绿灯 | PWM 输出 |
| 1 | 黄灯 | PWM 输出 |
| 2 | 红灯 | PWM 输出 |

有源低电平：PWM duty=0 时全亮，duty=255 时全灭。固件自动处理取反。

## PWM 参数

| 参数 | 值 |
|------|------|
| 频率 | 5000 Hz |
| 分辨率 | 8 位 (0-255) |
| 亮度范围 | 0=灭，255=最亮 |

## 支持的灯效模式

每个通道（绿/黄/红）可独立设置以下模式：

| 模式 | 值 | 说明 |
|------|------|------|
| OFF | 0 | 关闭 |
| SOLID | 1 | 常亮，亮度由 duty 控制 |
| BLINK | 2 | 闪烁，周期由 blink_period 控制 |
| BREATH | 3 | 呼吸（三角波），周期由 breath_period 控制 |

## 亮度控制

每个通道有独立的 duty 值（0-255），控制该通道的最大亮度：

- **常亮模式**：输出 = duty 值
- **闪烁模式**：亮时输出 duty，灭时输出 0
- **呼吸模式**：在 0 到 duty 之间做三角波

呼吸模式上升半周有感知整形（`lin²`），压低低占空比段斜率，补偿人眼对线性 PWM 的非线性感知。

## 同步控制

SET_MULTI 协议支持两种同步：

| 标志位 | 说明 |
|--------|------|
| blink_sync (bit0) | 所有 BLINK 模式的通道共享同一周期和相位 |
| breath_sync (bit1) | 所有 BREATH 模式的通道共享同一周期和相位 |

daemon 固定发送 `flags=0x03`（两个都同步）。

## 协议：SET_MULTI（cmd=1，16 字节）

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0-1 | 2 | magic | `0xA5 0x5A` |
| 2 | 1 | ver | 协议版本 = 2 |
| 3 | 1 | cmd | 1 = SET_MULTI |
| 4 | 1 | flags | bit0=blink_sync, bit1=breath_sync |
| 5-6 | 2 | blink_period | 闪烁周期 ms，uint16 LE，范围 50-60000 |
| 7-8 | 2 | breath_period | 呼吸周期 ms，uint16 LE，范围 50-60000 |
| 9 | 1 | mode_g | 绿灯模式: 0=OFF, 1=SOLID, 2=BLINK, 3=BREATH |
| 10 | 1 | mode_y | 黄灯模式 |
| 11 | 1 | mode_r | 红灯模式 |
| 12 | 1 | duty_g | 绿灯亮度 0-255 |
| 13 | 1 | duty_y | 黄灯亮度 |
| 14 | 1 | duty_r | 红灯亮度 |
| 15 | 1 | crc8 | CRC-8/ATM，计算范围 byte[2..14] |

## BLE 参数

| 参数 | 值 |
|------|------|
| 设备名 | VibeLight |
| Service UUID | `e52c12b6-7ac3-4636-9c17-3d608bcea796` |
| Characteristic UUID | `e52c12b7-7ac3-4636-9c17-3d608bcea796` |
| MTU | 128 |
| 广播间隔 | 100-200ms |

BLE 和串口共用同一套协议解析逻辑，可自由切换。

## 并发安全

- 环形缓冲区 (256 字节) 用 `portMUX` 临界区保护
- 灯态参数用独立的 `portMUX` 临界区保护
- PWM 刷新由 FreeRTOS 独立任务驱动（优先级 8，~1ms 周期）
- 协议解析与 PWM 刷新解耦，避免 BLE/串口阻塞导致灯效卡顿
