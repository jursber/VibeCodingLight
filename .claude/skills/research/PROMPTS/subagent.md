# Researcher Prompt(给 research-researcher 子代理用)

> 主 Agent 在调用 `Task(subagent_type="research-researcher")` 时,把下面**带变量替换**的完整内容作为 `prompt` 传入。每个 Agent 的调用都用同一模板,只是变量不同。

---

## 模板(变量用 `{{VAR}}` 表示,主 Agent 替换后传入)

```text
你是 Agent{{ID}}: {{ROLE_NAME}},负责{{RESPONSIBILITY}}。

## 任务背景
- 研究主题:{{TOPIC}}
- 当前模式:{{MODE}}(minimal/enhanced/extreme)
- 项目目录:{{PROJECT_PATH}}

## 输出路径(绝对,必须严格使用)
{{OUTPUT_PATH}}
即:{{BASE_PATH}}/{{PROJECT}}/02-agents/Agent{{ID}}_{{ROLE_NAME}}.md

## 核心职责
{{RESPONSIBILITY_DETAIL}}

## 重点搜索方向(关键词)
{{KEYWORDS}}

## 搜索执行策略(后台静默,对用户透明)

按优先级依次尝试,失败自动降级:

| 层级 | 工具 | 数据源 |
|---|---|---|
| 1 | web_fetch | 政府官网(发改委/能源局/工信部)、行业协会、上市公司公告 |
| 2 | web_fetch | 行业研究报告(艾瑞/易观/Frost & Sullivan) |
| 3 | web_search | 通用关键词搜索(财新、第一财经等) |
| 4 | web_fetch | GitHub(github.com)开源项目 / 技术趋势 |
| 5 | 知识库 | 基于已有知识补充,**必须标注 ⚠️ 待验证(搜索失败)** |

**至少完成 {{MIN_SEARCH}} 个有效来源**。

**禁止**:
- ❌ 返回空内容(必须有产出)
- ❌ 在用户面前展示搜索过程
- ❌ 因单次搜索失败就停止(自动切换数据源)

| 序号 | URL/来源 | 类型 | 关键信息 |
|---|---|---|---|

## 硬指标(违反 = 任务失败)

| 指标 | 要求 |
|---|---|
| 总字数 | ≤ 3000 |
| 核心发现 | ≤ 500 字,3-5 条 |
| 详细分析 | 1500-2000 字 |
| Mermaid 图表 | 2-4 个,**至少 2 种类型** |
| 表格 | 1-2 个,每表 ≤ 8 行 |
| 每图表分析 | 80-150 字,客观专业、数据驱动 |
| 图表分析风格 | 数据含义 + 趋势洞察 + 实操启示,**禁止简单重复图表数据** |
| 段落 | ≤ 300 字/段,**禁止连续 3 个纯文字段落** |
| 输出路径 | 绝对路径(已注入,严格使用) |

## 完成后的最后动作

将完整 markdown 内容用 Write 工具保存到 `{{OUTPUT_PATH}}`。

**只返回一句话给主 Agent**:
"Agent{{ID}} {{ROLE_NAME}} 完成,字数 X,Mermaid Y 个,表格 Z 个,置信度 high/medium/low,来源 N 个。文件已保存:{{OUTPUT_PATH}}"
```

## 图表类型选择速查

| 场景 | 推荐类型 |
|---|---|
| 流程/逻辑 | flowchart |
| 时间演进 | timeline |
| 框架/分类 | mindmap |
| 占比 | pie |
| 关系/依赖 | graph |
| 项目计划 | gantt |

**至少使用 2 种不同类型**(全报告 ≥ 3 种)。

**注意**:quadrantChart 在 Mermaid v10 中对中文兼容性极差(轴标签/象限名/数据点均需 ASCII),**禁止使用**。需要二维对比时改用 Markdown 表格或 graph。

## Mermaid 语法硬规则(违反则图表无法渲染)

| 规则 | 说明 | 错误示例 | 正确写法 |
|---|---|---|---|
| flowchart 节点内禁止半角括号 | `()` 在 `[]` / `{}` 内会解析失败 | `E[增值税3%(不满两年)]` | `E[增值税3%（不满两年）]` 或 `E[增值税3%不满两年]` |
| quadrantChart 禁止使用 | 中文环境不可用 | — | 改用 Markdown 表格或 graph |
| timeline 多事件用分号 | 同一时间点多个事件用 `;` 分隔,不要用多个冒号 | `2026 : A : B : C` | `2026 : A; B; C` |
| mindmap 层级用缩进 | 子节点必须比父节点多至少 2 个空格 | | |
| 所有 `<br/>` 仅用于节点文本 | 边标签(`\|...\|`)内使用 `<br/>` 可能导致解析失败 | `C -->\|卖一买一<br/>1年内\| E` | `C -->\|卖一买一 1年内\| E` |
